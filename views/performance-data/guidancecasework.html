Guidance Reasons casework can't be processed at first attempt

Many users need additional help to achieve their goal and providing this is an integral part of the service.

Collecting this data helps government discover unmet user needs and opportunities to reduce waste.

When a service has to contact users to progress their transactions the contacts (or a sample of them) are sorted into these categories:

User's information was illegible

This includes:

  • poor handwriting
  • user has made typo errors

User's information was incomplete

This includes:

  • there is no wet signature
  • the user has omitted key information
  • submitted with wrong or missing supporting information
  • government handling errors have resulted in missing information

User filled out the wrong form or used the wrong process

This includes:

  • user has applied for the wrong thing
  • the wrong form (outdated / wrong version) has been used
  • the user has attempted to complete several applications on one form

User's information didn't match government records

This includes:

  • data submitted by the user does not match government records
  • false information provided
